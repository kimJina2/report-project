# 회의록 작성 Skill

## 목적
`D:\report-project\skills\회의록\references\회의녹음\` 폴더의 STT 변환 텍스트 파일(*.txt)을 읽어
슈퍼컴퍼니 서비스개발1팀 공식 회의록 양식으로 정리하고,
Google Drive 회의록 폴더에 업로드한다.

회의록 양식: @references/meeting-template.md
작성 규칙 상세: @references/writing-rules.md

---

## 실행 순서

### Step 1 — STT 파일 확인

Filesystem MCP로 아래 폴더를 탐색한다:
`D:\report-project\skills\회의록\references\회의녹음\`

대상 파일 형식: `*_STT원본.txt` (STT 변환 텍스트 파일)

파일이 여러 개인 경우 아래 형식으로 목록을 보여주고 어느 파일을 처리할지 사용자에게 확인한다:

```
회의녹음 폴더에서 아래 STT 파일을 찾았습니다.

1. 20250401_회의_STT원본.txt (2025-04-01)
2. 20250406_회의_STT원본.txt (2025-04-06)

어느 파일의 회의록을 작성할까요? (번호 또는 파일명으로 답해주세요)
```

→ 파일이 1개면 사용자 확인 없이 바로 Step 2 진행.
→ 파일이 0개면 "회의녹음 폴더에 STT 파일이 없습니다"를 보고하고 중단.

---

### Step 2 — STT 텍스트 분석

선택된 txt 파일을 Filesystem MCP로 읽어 아래 항목을 추출한다.
추출 규칙 상세는 @references/writing-rules.md 참조.

추출 항목:
- 회의 일시 (파일명 또는 텍스트 내 언급 기준)
- 참석자 이름 및 역할
- 회의 목적 / 안건
- 안건별 논의 내용 요약
- 결정된 사항
- 액션아이템 (담당자 + 기한 포함)
- 다음 회의 일정 (언급된 경우)

분석 완료 후 사용자에게 보고:

```
STT 텍스트 분석 완료

파일: 20250406_회의_STT원본.txt
인식된 참석자: Thomas Moon, Daniel Kim, Ethan Lee, Michael Park
안건 수: 3개
액션아이템: 5건

회의록을 작성할까요?
```

→ 사용자가 확인하면 Step 3 진행.

---

### Step 3 — 회의록 작성 및 사용자 컨펌

@references/meeting-template.md 양식에 맞춰 회의록 초안을 작성한다.

**파일명 규칙:** `YYYYMMDD_[회의명].md`
- 날짜: STT 파일명의 날짜 부분 사용 (예: `20250406_회의_STT원본.txt` → `20250406`)
- 회의명: 텍스트에서 파악한 회의 목적 2~6글자 요약
- 예시: `20250406_팀주간회의.md`, `20250401_CloudWatch기획.md`

초안 작성 완료 후 반드시 아래 형식으로 사용자에게 컨펌을 요청한다.
이 단계는 오류·누락을 사전에 확인하기 위해 반드시 거쳐야 한다.
컨펌 없이 절대 Google Drive에 업로드하지 않는다.

```
회의록 초안이 완성됐습니다.

파일명: 20250406_팀주간회의.md
업로드 위치: Google Drive 회의록 폴더

--- 초안 미리보기 ---
[회의록 전체 내용 출력]
--------------------

수정할 내용이 있으신가요? (없으면 "업로드해줘" / 있으면 수정 내용을 말씀해주세요)
```

→ "업로드해줘", "올려줘", "괜찮아", "없어" → Step 4 진행
→ 수정 요청 → 내용 수정 후 미리보기 다시 출력, 재확인

---

### Step 4 — Google Drive 업로드

Google Drive MCP로 아래 폴더에 회의록 파일을 업로드한다:
`${GOOGLE_DRIVE_FOLDER_URL}`

업로드 완료 후 아래 형식으로 완료를 보고한다:

```
업로드 완료됐습니다.

파일명: 20250406_팀주간회의.md
위치: Google Drive 회의록 폴더
업로드 시각: YYYY-MM-DD HH:MM
링크: [Google Drive 파일 링크]
```

## 예외 처리

| 상황 | 처리 방법 |
|------|---------|
| STT 파일 0개 | 사용자에게 알리고 중단. 폴더 경로 및 파일 형식 재확인 안내 |
| txt 파일이 비어 있음 | "파일 내용이 없습니다"를 보고하고 중단 |
| 참석자 이름 불명확 | "확인 불가"로 표기 후 계속 진행. 컨펌 시 사용자에게 확인 요청 |
| 파악 불가한 구간 | "[내용 파악 불가]"로 표기 |
| 컨펌에서 수정 요청 | 수정 후 미리보기 재출력 → 재확인 |
| Google Drive 업로드 실패 | 오류 내용 보고 후 재시도 여부 확인 |

## Gotchas (자주 발생하는 실수)

- STT 텍스트에는 발화자 구분 없이 텍스트만 나열되는 경우가 많다. 문맥으로 발화자를 최대한 특정한다.
- 결정사항과 액션아이템을 혼동하지 않는다. 결정사항은 "~하기로 했다", 액션아이템은 "누가 언제까지 ~한다"이다.
- STT 특성상 오타·띄어쓰기 오류가 있을 수 있다. 문맥에 맞게 교정해서 회의록에 기재한다.
- 회의 시작 전 잡담, 종료 후 대화는 회의록에 포함하지 않는다.
- 액션아이템에 기한이 언급되지 않은 경우 "(기한 미정)"으로 표기하고 컨펌 시 사용자에게 확인 요청한다.
- 파일명에 특수문자( / \ : * ? " < > | )는 사용하지 않는다.
