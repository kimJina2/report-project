# 회의록 작성 규칙 — 상세

## 전체 원칙

- 어투: 격식체, 개조식 (문장 끝 "~함", "~결정함", "~예정")
- 길이: 안건당 3~7줄 요약. 전체 회의록은 A4 2장 이내 권장
- 발화자 특정: STT 텍스트의 문맥으로 발화자를 최대한 특정해서 기재
- 중립성: 사실만 기재. 의견·감정·평가 판단은 포함하지 않음
- STT 교정: 오타·띄어쓰기 오류·잘못된 받아쓰기는 문맥에 맞게 바로잡아 기재

---

## STT 텍스트 파일 구조 이해

STT 변환 파일은 아래 두 가지 형태 중 하나로 제공된다.

**형태 A — 발화자 구분 있음**
```
[Thomas Moon] 이번 주 CloudWatch 배포 일정 얘기해봅시다.
[Daniel Kim] 네, 이번 금요일로 잡고 있습니다.
```

**형태 B — 발화자 구분 없음 (연속 텍스트)**
```
이번 주 CloudWatch 배포 일정 얘기해봅시다. 네 이번 금요일로 잡고 있습니다. 
혹시 QA는 완료됐나요? 아직 진행 중입니다.
```

형태 B의 경우 문맥·역할·고유명사를 단서로 발화자를 추정한다.
특정이 불가능한 발화는 발화자 없이 내용만 요약한다.

## 섹션별 작성 규칙

### 회의명
- STT 텍스트 내 언급된 회의 목적으로 2~6글자 요약
- 파일명에 힌트가 있으면 우선 사용 (예: `20250406_회의_STT원본.txt` → 텍스트 내용으로 구체화)
- 예: "팀 주간회의", "CloudWatch 기획회의", "장애 대응 회의"

### 일시
- 파일명의 날짜 부분을 기준으로 기재 (예: `20250406` → 2025년 4월 6일)
- 회의 시작·종료 시각은 텍스트 내 언급된 경우만 기재. 없으면 "HH:MM ~ 미상"

### 참석자
- STT 텍스트에서 발화하거나 호명된 인물을 모두 기재
- 이름을 특정할 수 없는 참석자는 "참석자 A", "참석자 B"로 기재
- 소속·역할은 아는 경우만 기재. 모르면 "-"로 표기

### 논의 내용
- 안건별로 구분해서 작성
- STT 원문을 그대로 옮기지 않음. 핵심 내용만 요약
- 반복되거나 폐기된 의견은 제외
- 수치·날짜·고유명사(프로젝트명, 제품명 등)는 정확하게 기재
- STT 오인식으로 파악 불가한 내용: `[내용 파악 불가]`로 표기

### 결정사항
- STT 텍스트에서 "~하기로 했다", "~결정했다", "~확정했다" 패턴으로 도출
- 최종 합의·승인된 내용만 기재. 논의 중 폐기된 아이디어는 제외
- "~하기로 결정함" 형식으로 종결

### 액션아이템
- STT 텍스트에서 "~해주세요", "~담당해주시겠어요", "~까지 해주세요" 패턴으로 도출
- 형식: 담당자 + 할 일 + 기한
- 담당자 불명확 → "미정"으로 기재 후 컨펌 시 확인 요청
- 기한 미언급 → "(기한 미정)"으로 표기 후 컨펌 시 확인 요청
- 암묵적으로 합의된 액션아이템도 포함

## STT 발화자 특정 방법

| 단서 | 처리 방법 |
|------|---------|
| 이름 직접 언급 ("제가 JINA인데요") | 해당 이름으로 기재 |
| 역할로 언급 ("백엔드 쪽에서는") | 역할에 해당하는 팀원으로 추정 후 기재 |
| 호명 ("Ethan씨 어떻게 생각해요?") | 다음 발언 = Ethan으로 특정 |
| 전혀 특정 불가 | 발화자 없이 내용만 요약 |

## STT 오류 교정 기준

| STT 오류 유형 | 처리 방법 |
|-------------|---------|
| 명백한 오타 ("클라우드왓치" → "CloudWatch") | 올바른 표기로 교정 후 기재 |
| 받아쓰기 오류 ("배포를" → "배포를") | 문맥에 맞게 교정 |
| 의미 불명확한 단어 | 전후 문맥으로 추정. 불가하면 `[내용 파악 불가]` |
| 외래어·영어 혼용 | 원래 표기 유지 (예: PR, QA, API) |

## 파일명 규칙

형식: `YYYYMMDD_[회의명].md`

| 조건 | 처리 방법 |
|------|---------|
| STT 파일명에 날짜 포함 | 해당 날짜 사용 (예: `20250406_회의_STT원본.txt` → `20250406`) |
| 날짜 파악 불가 | 오늘 날짜 사용 |
| 회의명 파악 불가 | "회의"로 기재 (예: `20250406_회의.md`) |

파일명 금지 문자: `/ \ : * ? " < > |`
공백 대신 언더스코어(`_`) 사용
