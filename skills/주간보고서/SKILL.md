# 주간보고서 자동화 Skill

## 목적
매주 금요일, Gmail + Google Drive 소스를 수집해서
회사 양식 PPT를 완성하고 jn.kim 팀장에게 이메일로 발송한다.

슬라이드 매핑 규칙: @references/slide-mapping.md
이메일 발송 양식: @references/email-template.md

---

## 실행 순서

### Step 1 — 소스 수집

아래 3가지 소스를 순서대로 수집한다.

**(1) Gmail MCP — 이번 주 업무 관련 메일**
- 수집 기간: 직전 월요일 00:00 ~ 오늘(금요일) 현재 시각
- 수집 대상: fastkjn1@gmail.com 수발신 메일 전체
- 키워드 필터: 업무 관련 메일 (noreply, 광고, 스팸 제외)
- 제외: noreply, 자동발송, 광고, 구독 메일

**(2) Google Drive MCP — 회의록 폴더**
- 폴더: ${GOOGLE_DRIVE_FOLDER_URL}
- 이번 주(월~금) 생성·수정된 파일만 읽기

**(3) Google Drive MCP — 업무 결과물 폴더**
- 폴더: ${GOOGLE_DRIVE_FOLDER_URL}
- 이번 주(월~금) 생성·수정된 파일만 읽기

수집 완료 후 반드시 아래 형식으로 사용자에게 보고하고 진행 여부를 확인한다:

```
소스 수집 완료

Gmail: N건
  - [발신자] 제목 (날짜)
  - ...

회의록: N건
  - [파일명] (날짜)
  - ...

업무 결과물: N건
  - [파일명] (날짜)
  - ...

보고서 작성을 시작할까요?
```

→ 사용자가 확인하면 Step 2로 진행. 소스가 전혀 없으면 사용자에게 알리고 중단.

---

### Step 2 — 내용 분류

수집한 소스를 슬라이드별로 분류한다. 상세 규칙은 @references/slide-mapping.md 참조.

분류 기준 요약:
- 완료된 업무, 배포·릴리즈 내역 → 슬라이드 2 (주요 성과)
- 버그, 장애, 지연, 기술적 문제와 그 대응 → 슬라이드 3 (이슈 및 대응)
- 다음 주 예정 업무, 액션아이템 → 슬라이드 4 (다음 주 계획)
- 여러 소스에 같은 내용이 중복되면 한 번만 기재
- 소스 없는 슬라이드는 "해당 없음"으로 표기

---

### Step 3 — PPT 작성

**준비:**
- 양식 파일 복사: `D:\report-project\week_pptx\ppt_template\주간보고서.pptx`
- 저장 위치: `D:\report-project\week_pptx\weekly_report\`
- 파일명: `주간보고서_YYYYMMDD_v1.pptx`
- IMPORTANT: 원본 양식 파일 덮어쓰기 절대 금지. 반드시 복사본에 작업한다.

**작성 규칙:**
- @skills/pptx/SKILL.md 를 활용해 PPT를 생성한다
- 슬라이드 추가·삭제·순서 변경 금지
- 디자인·레이아웃·폰트·색상 변경 금지, 텍스트만 채운다
- 어투: 격식체
- 슬라이드당 3줄 이내로 간결하게
- 슬라이드별 상세 작성 규칙은 @references/slide-mapping.md 참조

---

### Step 4 — 사용자 컨펌 요청

PPT 완성 후 반드시 아래 형식으로 정확히 물어본다.
컨펌 없이 절대 발송하지 않는다. 이 단계는 자동 발송 사고를 방지하기 위해 반드시 거쳐야 한다.

```
보고서가 완성됐습니다.

파일: 주간보고서_YYYYMMDD_v1.pptx
수신자: jn.kim <fastkjn1@gmail.com>
제목: [주간보고] YYYY년 MM월 N주차 업무 요약

발송할까요? (네 / 아니요)
```

→ "네", "응", "보내줘", "발송해줘" → Step 5 진행
→ "아니요", 수정 요청 → 요청 내용 수정 후 파일명 버전 올려 재저장(v2, v3...), Step 4 재실행

---

### Step 5 — 이메일 발송

발송 양식은 @references/email-template.md 참조.

발송 완료 후 아래 형식으로 완료를 보고하고, 파일을 보관 폴더로 이동한다:

```
발송 완료되었습니다.

수신자: jn.kim <fastkjn1@gmail.com>
발송 시각: YYYY-MM-DD HH:MM
파일: D:\report-project\week_pptx\weekly_report\발송완료\주간보고서_YYYYMMDD_v1.pptx
```

## 예외 처리

| 상황 | 처리 방법 |
|------|---------|
| 소스 3종 모두 0건 | 사용자에게 알리고 중단. 수동 입력 여부 확인 |
| Gmail만 0건 | "이번 주 관련 메일이 없습니다"로 표기 후 나머지 소스로 계속 진행 |
| Google Drive 파일 읽기 실패 | 오류 내용 보고 후 해당 소스 제외하고 진행 여부 확인 |
| PPT 작성 실패 | 오류 내용 보고 후 중단. 수동 작성 안내 |
| 컨펌에서 "아니요" | 수정 요청 내용 확인 → 수정 → 파일명 버전 올려 재저장 → Step 4 재실행 |
| 이메일 발송 실패 | 오류 내용 보고 후 재시도 여부 확인. 재시도 시 같은 파일 재사용 |

## Gotchas (자주 발생하는 실수)

- 같은 이슈가 회의록과 이메일 양쪽에 나올 수 있다. 슬라이드에는 한 번만 기재한다.
- "검토 중", "논의 필요" 같은 미완료 항목은 슬라이드 2(주요 성과)가 아닌 슬라이드 4(다음 주 계획)에 넣는다.
- 파일명의 날짜는 보고서 작성일(금요일) 기준이다. 소스 파일 날짜가 아니다.
- Step 4 컨펌 메시지에서 수신자 이메일 주소를 반드시 표시한다. 빠뜨리면 사용자가 확인할 수 없다.
